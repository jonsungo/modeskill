# UI Consistency Methodology

Analyze consistency at five levels:

1. Semantic consistency: roles, meaning, hierarchy, state, and accessibility.
2. Component consistency: variants, sizes, composition, and reusable behavior.
3. Interaction consistency: keyboard, focus, dismissal, validation, and responsive state changes.
4. Engineering consistency: tokens, naming, shared primitives, structure, and maintainability.
5. Visual and brand consistency: typography, color, spacing, density, shape, and approved brand variation.

Dimensions include typography, color semantics, spacing, components, forms, modal behavior, responsive behavior, and engineering conventions.

## Process

1. Confirm the Workspace root and read-only discovery scope.
2. Validate selected reference and target projects against current discovery/manual projects.
3. Inventory centralized sources before local overrides.
4. Compare reference intent and target implementation across all five levels.
5. Record structured evidence with stable locators.
6. Classify conclusions and mark agent inference as `inferred` evidence.
7. Put weak or conflicting claims under `unresolved`.
8. Produce a Profile or report without treating visual identity as the goal.

Do not infer a global rule from one incidental CSS value.
