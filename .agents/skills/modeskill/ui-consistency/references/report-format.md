# Consistency Report Format

Include:

1. Reference sources.
2. Project Profile draft.
3. Inherited rules.
4. Adapted rules.
5. Project-specific rules.
6. Unresolved conflicts.
7. Deprecated patterns.
8. Implementation recommendations.
9. Files that would be affected.
10. Risk level and write-authorization status.
11. Known limits.

Every conclusion requires classification, dimension, title, rationale, confidence, and structured evidence. Summary counts must match actual conclusions. Keep recommendations separate from execution and state when the task remains read-only.
