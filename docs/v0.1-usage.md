# Modeskill v0.1 使用说明

## 设置 Workspace

1. 运行 `python3 .agents/skills/modeskill/configurator/server.py`。
2. 打开 `http://127.0.0.1:8765`；页面默认中文，右上角可切换 English。
3. 输入 Workspace root，点击“重新扫描项目”。
4. 从自动发现和手动登记的项目中选择主要参考、辅助参考和目标项目。
5. 验证后保存到 `.local/workspaces/`。
6. 使用结束后在启动服务的终端按 `Control+C`。

字段旁的 `?` 按钮用于解释技术术语。可按 Tab 聚焦，使用 Enter 或 Space 打开说明，按 Esc 关闭。

发现缓存可以随时刷新。项目删除或改名后，失效的角色选择会报错。Workspace root 是只读边界，不是操作系统授权；无法读取时会明确报告环境权限错误。

## 理解权限

- 参考项目永远只读。
- 目标项目默认只读。
- 配置允许 `explicit-per-task` 只代表可以发起申请。
- 真正写入还需要当前任务明确授权和运行环境实际写权限。
- 设置界面不能创建或保存当前任务授权。

## 理解模块分发

- “与 Git 仓库同步”表示模块位于受 Git 管理的 Skill 目录。设置界面不会自动 commit 或 push。
- “仅保留在本机”表示模块位于 `.local/modules` 并被 Git 忽略。
- 当前策略固定，不提供自由迁移，也不能修改已有 Git 历史。

本机模块和 Workspace 配置不会上传 GitHub，换电脑或重新 Clone 后不会自动回来。删除仓库前必须备份 `.local/modules` 和 `.local/workspaces`。可在设置界面复制本机模块目录路径。

本地服务只接受本机 Host。浏览器 POST 必须来自同一 `127.0.0.1` 或 `localhost` 地址；无 `Origin` 请求只允许本机回环客户端。所有 POST 必须是 1 MB 内的 JSON 对象。

## 使用可选本机能力

在“功能模块”区域确认本机模块已安装并启用，然后调用：

```text
使用 $modeskill，用大白话解释当前项目的技术逻辑。
使用 $modeskill，生成适合求职面试的三句话技术介绍。
```

如果模块或仓库不可访问，Modeskill 会提示：`本机模块未安装或当前 Modeskill 仓库不可访问。`

v0.1 的 UI 项目发现只列出项目和基础元数据，不执行完整自动扫描。自动提取、漂移检测和受控写入仍属于后续版本。
